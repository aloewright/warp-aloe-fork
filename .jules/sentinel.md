## 2025-05-14 - [HIGH] Command injection in CLI installation via AppleScript
**Vulnerability:** Command injection vulnerability in the CLI installation and uninstallation logic on macOS. The application used `osascript` to run `do shell script` with administrator privileges. It attempted to escape paths using POSIX shell escaping, but embedded these strings directly into an AppleScript string literal. A malicious path containing double quotes or backslashes could break out of the AppleScript literal and execute arbitrary commands with elevated privileges.
**Learning:** AppleScript's `do shell script` involves two layers of parsing: first by the AppleScript interpreter itself (handling the string literal), and then by `/bin/sh`. Using POSIX shell escaping from the host language is insufficient and dangerous because it doesn't account for AppleScript's own string syntax.
**Prevention:** Always use AppleScript's native `quoted form of` property when passing parameters to `do shell script`. Additionally, properly escape the string for AppleScript literals (escaping `"` and `\`) before embedding it in the script passed to `osascript`.
